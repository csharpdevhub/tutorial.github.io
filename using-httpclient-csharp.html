Using HttpClient in C#: Complete Guide to Calling REST APIs

Published: June 2026 | Category: C# Networking | Reading Time: 12 Minutes

Most modern applications communicate with external services through APIs. Whether you're retrieving weather information, processing payments, authenticating users, or integrating third-party services, HttpClient is the primary tool used in .NET for sending HTTP requests.

In this guide, you'll learn how to use HttpClient to perform GET, POST, PUT, and DELETE requests, handle JSON responses, configure headers, and follow recommended best practices.

What Is HttpClient?

HttpClient is a class provided by .NET that allows applications to send HTTP requests and receive responses from web servers.

It supports common HTTP methods such as:

  • GET
  • POST
  • PUT
  • PATCH
  • DELETE

HttpClient is commonly used when interacting with REST APIs, microservices, cloud platforms, and external web applications.

Creating an HttpClient

The simplest way to create an HttpClient instance is: 


        using System.Net.Http;

        HttpClient client = new HttpClient();

Once created, the client can send requests to remote servers.

Making a GET Request

GET requests retrieve data from an API. 


        using System.Net.Http;

        HttpClient client = new HttpClient();

        string response =
        await client.GetStringAsync(
        "https://jsonplaceholder.typicode.com/posts/1");

        Console.WriteLine(response);

Output


        {
        "userId": 1,
        "id": 1,
        "title": "...",
        "body": "..."
        }

Using GetAsync()

GetAsync() provides more control over the response. 


        HttpResponseMessage response =
        await client.GetAsync(
        "https://jsonplaceholder.typicode.com/posts/1");

        if (response.IsSuccessStatusCode)
        {
        string content =
        await response.Content.ReadAsStringAsync();

        Console.WriteLine(content);
        }

This approach allows access to status codes, headers, and other response information.

Sending a POST Request

POST requests send data to a server. 


        using System.Text;
        using System.Net.Http;

        string json =
        """
        {
        "title":"New Post",
        "body":"Example Content"
        }
        """;

        StringContent content =
        new StringContent(
        json,
        Encoding.UTF8,
        "application/json");

        HttpResponseMessage response =
        await client.PostAsync(
        "https://jsonplaceholder.typicode.com/posts",
        content);

This creates a new resource on the target server.

Reading JSON Responses

Most APIs return JSON data.

The System.Text.Json namespace makes deserialization straightforward. 


        using System.Text.Json;

        var post =
        JsonSerializer.Deserialize<Post>(jsonResponse);

Example Model


        public class Post
        {
        public int Id { get; set; }

        public string Title { get; set; }

        public string Body { get; set; }
        }

Sending Custom Headers

Many APIs require authentication headers. 


        client.DefaultRequestHeaders.Add(
        "Authorization",
        "Bearer YOUR_TOKEN");

Headers can also include API keys, custom identifiers, and content negotiation settings.

PUT Requests

PUT requests update existing resources. 


        HttpResponseMessage response =
        await client.PutAsync(
        url,
        content);

DELETE Requests

DELETE requests remove resources from a server. 


        HttpResponseMessage response =
        await client.DeleteAsync(
        "https://api.example.com/posts/1");

Error Handling

Network failures and server errors should always be handled properly. 


        try
        {
        HttpResponseMessage response =
        await client.GetAsync(url);

        response.EnsureSuccessStatusCode();
        }
        catch (HttpRequestException ex)
        {
        Console.WriteLine(ex.Message);
        }

This helps applications recover gracefully when requests fail.

Using HttpClientFactory in ASP.NET Core

In ASP.NET Core applications, Microsoft recommends using HttpClientFactory.

It manages HttpClient lifetimes and prevents common issues such as socket exhaustion. 


        builder.Services.AddHttpClient();

Services can then receive HttpClient through dependency injection. 


        public class WeatherService
        {
        private readonly HttpClient _client;

        public WeatherService(HttpClient client)
        {
        _client = client;
        }
        }

Common Mistakes

  • Creating a new HttpClient for every request.
  • Ignoring exceptions.
  • Not validating API responses.
  • Blocking async code using .Result.
  • Hardcoding API credentials.

Best Practices

  • Use async and await.
  • Prefer HttpClientFactory in ASP.NET Core.
  • Reuse HttpClient instances when possible.
  • Handle failures gracefully.
  • Use strongly typed models for JSON data.
  • Store API keys securely.

Frequently Asked Questions

Is HttpClient thread-safe?

Yes. HttpClient is designed to be reused across multiple requests.

Should I dispose HttpClient?

In ASP.NET Core applications, HttpClientFactory handles this automatically. For standalone applications, reuse instances whenever possible.

Can HttpClient call REST APIs?

Yes. HttpClient is one of the most common tools used to consume REST APIs in .NET.

Why use async with HttpClient?

Async operations prevent threads from blocking while waiting for network responses.
